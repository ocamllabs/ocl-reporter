OPAM

OPAM is the Ocaml PAckage Manager and its features include: powerful handling of dependencies (versions constraints, optional dependencies, conflicts, etc), can deal with multiple repositories' backends (HTTP, rsync, git) and the ability to switch between different OCaml compiler versions.

The fine grained handling of dependencies is made possible by the use of the CUDF library and the dose3 solver developed by the Mancoosi project, which is, among other things, used by Debian to manage their packages

In brief, if you use OCaml, OPAM will probably make your life easier. Check out opam.ocamlpro.com for installation instructions and package information.

The State of OPAM

Talk on The State of OPAM at OCaml Users and Developers Workshop 2012:

The OCaml ecosystem is still lacking a good package manager. Over the last few years, several attempts have been made to try to fix this issue, but they all suffer the same drawbacks: they all make basic assumptions which are convenient when you deal with a small number of packages, but which make them do not scale because they do not handle the dependencies and conflicts correctly.

  • Slides
  • Abstract
  • Video

14 Sep 2012: Thomas Gazagnaire explained the design rationale behind OPAM, and why we need yet another package manager. Audience questions were generally positive, but several people expressed concern that the build, packaging and test systems were all diverging. OPAM aims to unify them eventually, but starts by wrapping existing systems for compatibility.

opam

OPAM is a source-based package manager for OCaml. It supports multiple simultaneous compiler installations, flexible package constraints, and a Git-friendly development workflow.
